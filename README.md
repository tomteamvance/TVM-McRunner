# McRun Orientation Game

A mobile-first endless runner for orientation waiting areas.

## Included

- Responsive phone gameplay
- Tap and double-jump controls
- Animated crew character and restaurant street scene
- Obstacles, collectibles, particles, score multipliers, and shields
- Sound effects with mute control
- Optional nickname and store number
- Local leaderboard saved on the device
- Optional Supabase-powered global leaderboard
- No player-facing button or menu for clearing scores

## Put it on GitHub Pages

1. Create a public GitHub repository, such as `mcrun`.
2. Upload `index.html`, `styles.css`, `game.js`, and `config.js` to the repository root.
3. Open **Settings → Pages**.
4. Choose **Deploy from a branch**.
5. Select the `main` branch and `/root`, then save.
6. Your address will look like `https://YOUR-USERNAME.github.io/mcrun/`.

## Turn on the global leaderboard

The game works immediately with a local leaderboard. For one shared leaderboard across every phone, create a free Supabase project.

### 1. Create the table

Open Supabase **SQL Editor** and run:

```sql
create table public.scores (
  id bigint generated by default as identity primary key,
  player_name text not null check (char_length(player_name) between 1 and 16),
  store_number text null check (char_length(store_number) <= 8),
  score integer not null check (score >= 0 and score <= 10000000),
  created_at timestamptz not null default now()
);

alter table public.scores enable row level security;

create policy "Anyone can read scores"
on public.scores for select
using (true);

create policy "Anyone can submit scores"
on public.scores for insert
with check (
  score >= 0
  and score <= 10000000
  and char_length(player_name) between 1 and 16
  and (store_number is null or char_length(store_number) <= 8)
);

create index scores_score_idx on public.scores (score desc, created_at asc);
```

### 2. Add the project values

Open `config.js` and paste your Supabase values:

```js
window.MCRUN_CONFIG = {
  supabaseUrl: "https://YOUR-PROJECT.supabase.co",
  supabaseAnonKey: "YOUR-ANON-KEY",
  tableName: "scores",
  maxGlobalRows: 50
};
```

The anon key is intended for browser use. Keep the Supabase service-role key private and never place it in this project.

## QR code

After GitHub Pages gives you the final URL, create a QR code that points to that exact address. The QR code should not point to the ZIP file or GitHub repository page.

## Score behavior

- Device scores are stored in browser local storage.
- Global scores are stored in Supabase when configured.
- There is intentionally no score-clearing control in the player interface.
- A technically knowledgeable person can still clear their own browser storage from browser settings. Global Supabase scores remain unless an administrator deletes them in Supabase.

## Score milestones

During each run, the game displays a one-time celebration when the player reaches:

- 50,000 points — **Milestone Unlocked**
- 100,000 points — **Legendary Run**

These celebrations do not erase or modify leaderboard scores.
